# user-profile-info

## 目的

用户个人信息展示功能，在 iOS 个人中心页面动态加载并显示用户的头像、用户名和个人签名。支持响应账号切换和自动刷新用户资料。

## 需求

### 需求：显示用户头像
系统必须在个人信息卡片中显示当前登录用户的头像。头像必须从账号管理器获取并使用 URL 加载图片。

#### 场景：成功加载头像
- **当** 用户已登录且头像 URL 有效
- **那么** 系统必须显示用户的头像图片

#### 场景：使用默认头像
- **当** 用户已登录但头像 URL 为空或无效
- **那么** 系统必须显示默认的用户图标（person.fill）

#### 场景：未登录状态
- **当** 用户未登录
- **那么** 系统必须显示默认的用户图标

### 需求：显示用户名
系统必须在个人信息卡片中显示当前登录用户的用户名。

#### 场景：显示已登录用户名
- **当** 用户已登录
- **那么** 系统必须显示该用户的用户名

#### 场景：显示未登录提示
- **当** 用户未登录
- **那么** 系统必须显示"未登录"文本

### 需求：显示个人签名
系统必须在个人信息卡片中显示用户的个人签名。签名文本必须限制在 2 行内显示。

#### 场景：显示个人签名
- **当** 用户已登录且有个人签名
- **那么** 系统必须显示个人签名文本，最多显示 2 行

#### 场景：签名为空
- **当** 用户已登录但个人签名为空
- **那么** 系统必须显示默认提示文本

### 需求：响应账号更新
系统必须监听账号管理器的更新通知，并在账号信息变化时自动刷新显示的用户信息。

#### 场景：账号切换时更新
- **当** 用户切换到另一个账号
- **那么** 系统必须自动更新显示的头像、用户名和签名

#### 场景：账号登出时更新
- **当** 用户登出账号
- **那么** 系统必须清除显示的用户信息并显示未登录状态

### 需求：自动刷新用户数据
系统必须在页面加载时自动从服务器刷新用户的最新信息。

#### 场景：页面加载时刷新
- **当** 用户进入个人中心页面
- **那么** 系统必须调用账号管理器刷新当前用户的最新资料

#### 场景：刷新失败处理
- **当** 刷新用户资料失败
- **那么** 系统必须继续显示本地缓存的用户信息
- **并且** 不应阻止用户使用其他功能

### 需求：显示二维码入口
系统必须在个人信息卡片中提供二维码功能的入口按钮。

#### 场景：显示二维码按钮
- **当** 用户已登录
- **那么** 系统必须显示二维码图标按钮

#### 场景：点击二维码按钮
- **当** 用户点击二维码图标
- **那么** 系统必须展示用户的个人二维码信息
