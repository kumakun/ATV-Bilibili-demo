# ipad-profile-splitview

## 目的

在 iPad 上为个人中心页面提供 NavigationSplitView 交互方式，实现侧边栏功能列表与详情页内容的联动显示，提升大屏幕设备上的浏览效率和用户体验。

## 新增需求

### 需求：iPad 设备检测
系统必须能够检测当前运行的设备类型，以决定使用 NavigationSplitView 还是 NavigationStack 布局方式。

#### 场景：识别 iPad 设备
- **当** 应用在 iPad 设备上运行
- **那么** 系统必须使用 NavigationSplitView 布局

#### 场景：识别 iPhone 设备
- **当** 应用在 iPhone 设备上运行
- **那么** 系统必须使用 NavigationStack 布局

### 需求：侧边栏内容展示
系统必须在 iPad 的 NavigationSplitView 侧边栏中展示用户信息卡片和功能列表。

#### 场景：显示用户信息卡片
- **当** 用户在 iPad 上打开个人中心页面
- **那么** 系统必须在侧边栏顶部显示用户信息卡片
- **并且** 用户信息卡片必须包含头像、用户名、个人签名和二维码入口

#### 场景：显示功能列表
- **当** 用户在 iPad 上打开个人中心页面
- **那么** 系统必须在侧边栏中显示功能列表
- **并且** 功能列表必须包含：关注UP、追番追剧、历史记录、稍后再看、每周必看、账号切换、登出

#### 场景：列表项显示样式
- **当** 侧边栏展示功能列表时
- **那么** 每个功能项必须显示图标、文字和颜色标识
- **并且** 列表项必须支持选中状态的视觉反馈

### 需求：功能项选择和导航
系统必须支持用户在侧边栏中选择功能项，并在详情页区域显示对应的内容。

#### 场景：选择功能项显示详情
- **当** 用户在侧边栏中点击某个功能项（如"关注UP"）
- **那么** 系统必须在详情页区域显示对应的功能页面
- **并且** 侧边栏中该功能项必须显示为选中状态

#### 场景：切换不同功能项
- **当** 用户在侧边栏中选择另一个功能项
- **那么** 系统必须立即在详情页区域切换到新的功能页面
- **并且** 侧边栏的选中状态必须更新为新选择的功能项

#### 场景：首次进入时无选中项
- **当** 用户首次在 iPad 上打开个人中心页面
- **那么** 系统禁止默认选中任何功能项
- **并且** 详情页区域必须显示占位提示（如"请选择左侧功能"）

### 需求：详情页内容展示
系统必须在 NavigationSplitView 的详情页区域展示选中功能的完整内容。

#### 场景：展示功能详情页
- **当** 用户选中某个功能项
- **那么** 系统必须在详情页区域显示该功能对应的完整视图
- **并且** 详情页必须占据侧边栏右侧的所有可用空间

#### 场景：详情页支持独立导航
- **当** 功能详情页包含子页面导航（如视频详情）
- **那么** 系统必须支持在详情页区域内进行 push 导航
- **并且** 侧边栏必须保持可见和交互

### 需求：登出确认对话框
系统必须支持在侧边栏中触发登出操作，并显示确认对话框。

#### 场景：点击登出显示确认对话框
- **当** 用户在侧边栏中点击"登出"功能项
- **那么** 系统必须显示登出确认对话框
- **并且** 对话框必须包含"登出"和"取消"两个选项

#### 场景：确认登出
- **当** 用户在确认对话框中点击"登出"
- **那么** 系统必须执行登出操作
- **并且** 登出完成后必须更新侧边栏的用户信息显示

#### 场景：取消登出
- **当** 用户在确认对话框中点击"取消"
- **那么** 系统必须关闭对话框并保持登录状态

### 需求：侧边栏宽度和显示模式
系统必须根据 iPad 屏幕尺寸和方向自适应调整侧边栏的宽度和显示模式。

#### 场景：正常屏幕尺寸的双栏显示
- **当** iPad 屏幕宽度足够（如 iPad Pro、iPad Air 横屏）
- **那么** 系统必须同时显示侧边栏和详情页（双栏模式）

#### 场景：小屏幕设备的自动收起
- **当** iPad 屏幕宽度较小（如 iPad mini 竖屏）
- **那么** 系统必须支持侧边栏自动收起为 overlay 模式
- **并且** 用户必须可以通过手势或按钮打开侧边栏

### 需求：状态保持和恢复
系统必须保持和恢复用户在侧边栏中的选择状态。

#### 场景：切换页面后返回
- **当** 用户从个人中心页面切换到其他 Tab，然后返回
- **那么** 系统必须保持之前选中的功能项状态
- **并且** 详情页必须显示之前的内容

#### 场景：应用重启后恢复
- **当** 应用重启后用户打开个人中心页面
- **那么** 系统禁止自动恢复之前的选中状态
- **并且** 必须显示未选中任何功能项的初始状态

### 需求：账号切换功能集成
系统必须支持在侧边栏中触发账号切换功能，并在切换后更新侧边栏内容。

#### 场景：打开账号切换器
- **当** 用户在侧边栏中点击"账号切换"功能项
- **那么** 系统必须以模态方式显示账号切换器页面

#### 场景：账号切换后更新
- **当** 用户在账号切换器中切换到另一个账号
- **那么** 系统必须更新侧边栏中的用户信息卡片
- **并且** 系统必须清除当前选中的功能项状态
- **并且** 详情页必须返回到未选中的初始状态
