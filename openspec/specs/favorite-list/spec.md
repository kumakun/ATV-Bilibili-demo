## 新增需求

### 需求：展示用户的收藏夹列表
系统必须展示当前登录用户的所有收藏夹，包括用户自建的收藏夹和用户收藏的他人订阅收藏夹。

#### 场景：用户已登录且有收藏夹
- **当** 用户打开收藏页面
- **那么** 系统必须显示收藏夹列表，每个收藏夹显示名称、图标和内容数量

#### 场景：用户已登录但无收藏夹
- **当** 用户打开收藏页面且该用户没有任何收藏夹
- **那么** 系统必须显示空状态提示信息

#### 场景：用户未登录
- **当** 未登录用户打开收藏页面
- **那么** 系统必须显示登录引导界面

### 需求：区分自建收藏夹和订阅收藏夹
系统必须在 UI 上区分用户自建的收藏夹和订阅的他人收藏夹。

#### 场景：展示自建收藏夹
- **当** 收藏夹为用户自己创建的
- **那么** 系统必须使用标准文件夹图标展示

#### 场景：展示订阅收藏夹
- **当** 收藏夹为用户收藏的他人订阅
- **那么** 系统必须使用不同颜色或标识区分（例如使用特殊图标或标签）

### 需求：处理收藏夹加载错误
系统必须妥善处理收藏夹列表加载失败的情况。

#### 场景：网络请求失败
- **当** 收藏夹列表加载因网络错误失败
- **那么** 系统必须显示错误信息和重试按钮

#### 场景：API 返回错误
- **当** API 返回错误状态码或无效数据
- **那么** 系统必须显示友好的错误提示信息

### 需求：点击收藏夹进入内容页
系统必须支持用户点击收藏夹查看其中的内容。

#### 场景：用户点击收藏夹
- **当** 用户点击收藏夹卡片
- **那么** 系统必须导航到该收藏夹的内容展示页面

### 需求：显示加载状态
系统必须在加载收藏夹列表时提供视觉反馈。

#### 场景：首次加载
- **当** 用户首次打开收藏页面
- **那么** 系统必须显示加载指示器直到数据加载完成

#### 场景：下拉刷新
- **当** 用户执行下拉刷新操作
- **那么** 系统必须显示刷新指示器并重新加载收藏夹列表
