# weekly-watch

## 目的

每周必看功能，在 iOS 端展示 B 站的每周必看推荐视频列表，自动加载最新一期内容，展示期号标题和视频列表。

## 需求

### 需求：显示每周必看列表
系统必须显示 B 站的每周必看推荐视频列表，包括最新一期的标题和视频内容。

#### 场景：加载每周必看
- **当** 用户打开每周必看页面
- **那么** 系统必须从服务器获取每周必看系列列表
- **并且** 自动加载最新一期的视频内容

#### 场景：加载失败
- **当** 获取每周必看数据失败
- **那么** 系统必须显示错误提示
- **并且** 允许用户重试

### 需求：显示期号标题
系统必须在页面顶部显示当前加载的每周必看期号和主题标题。

#### 场景：显示标题信息
- **当** 加载最新一期每周必看
- **那么** 系统必须在顶部显示期号名称（如"每周必看第 123 期"）
- **并且** 显示该期的主题标题

### 需求：显示视频列表
系统必须显示该期每周必看包含的所有推荐视频。

#### 场景：显示视频卡片
- **当** 加载视频列表成功
- **那么** 系统必须显示每个视频的封面图
- **并且** 显示视频标题
- **并且** 显示 UP 主信息
- **并且** 显示视频统计数据

### 需求：自动加载最新一期
系统必须默认加载并显示最新一期的每周必看内容。

#### 场景：首次进入页面
- **当** 用户首次打开每周必看页面
- **那么** 系统必须获取每周必看系列列表
- **并且** 自动选择列表中的第一期（最新一期）
- **并且** 加载该期的视频内容

### 需求：禁用下拉刷新
系统禁止在每周必看页面使用下拉刷新功能。

#### 场景：页面加载
- **当** 用户打开每周必看页面
- **那么** 系统必须自动加载数据

#### 场景：尝试下拉刷新
- **当** 用户尝试下拉刷新
- **那么** 系统禁止触发刷新操作

### 需求：播放视频
系统必须支持用户点击视频卡片后播放该视频。

#### 场景：点击视频
- **当** 用户点击某个视频卡片
- **那么** 系统必须打开视频详情页面
- **并且** 开始播放该视频

### 需求：优化列表布局
系统必须使用侧边栏样式的列表布局展示每周必看视频列表，适配 iOS 设备。

#### 场景：iOS 设备显示
- **当** 在 iOS 设备上显示每周必看列表
- **那么** 系统必须使用侧边栏样式布局
- **并且** 在顶部显示标题信息
- **并且** 保证视频列表清晰可读

### 需求：显示标题头部
系统必须在视频列表上方显示固定的期号和主题标题头部。

#### 场景：显示头部
- **当** 视频列表加载完成
- **那么** 系统必须在列表顶部显示标题头部
- **并且** 头部包含期号名称和主题
