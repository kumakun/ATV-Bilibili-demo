# watch-later

## 目的

稍后再看功能，在 iOS 端展示用户添加的稍后再看视频列表，支持显示视频详细信息、时长格式化、删除视频和播放功能。

## 需求

### 需求：显示稍后再看列表
系统必须显示用户添加到"稍后再看"的所有视频，包括封面图、标题、UP 主、时长和统计信息。

#### 场景：加载稍后再看列表
- **当** 用户打开稍后再看页面
- **那么** 系统必须从服务器获取稍后再看列表
- **并且** 显示所有视频的详细信息

#### 场景：加载失败
- **当** 获取稍后再看列表失败
- **那么** 系统必须显示错误提示
- **并且** 允许用户重试

#### 场景：空列表
- **当** 稍后再看列表为空
- **那么** 系统必须显示空状态提示

### 需求：显示视频详细信息
系统必须在每个视频卡片上显示完整的视频信息。

#### 场景：显示基本信息
- **当** 显示视频卡片
- **那么** 系统必须显示视频封面图
- **并且** 显示视频标题
- **并且** 显示 UP 主名称和头像
- **并且** 显示视频时长
- **并且** 显示发布时间

#### 场景：显示统计数据
- **当** 视频有统计数据
- **那么** 系统必须显示播放量
- **并且** 显示弹幕数量

### 需求：格式化视频时长
系统必须将视频时长格式化为可读的时间格式。

#### 场景：短视频时长
- **当** 视频时长小于 1 小时
- **那么** 系统必须显示为"分:秒"格式（如"12:34"）

#### 场景：长视频时长
- **当** 视频时长大于等于 1 小时
- **那么** 系统必须显示为"时:分:秒"格式（如"1:23:45"）

### 需求：播放视频
系统必须支持用户点击视频卡片后播放该视频。

#### 场景：点击视频
- **当** 用户点击某个视频卡片
- **那么** 系统必须打开视频详情页面
- **并且** 传递视频的 aid 和 cid 参数

### 需求：删除视频
系统必须支持用户从稍后再看列表中删除视频。

#### 场景：长按删除
- **当** 用户长按某个视频卡片
- **那么** 系统必须显示删除确认对话框

#### 场景：确认删除
- **当** 用户确认删除操作
- **那么** 系统必须调用删除 API 删除该视频
- **并且** 刷新列表移除该视频

#### 场景：取消删除
- **当** 用户取消删除操作
- **那么** 系统必须关闭对话框
- **并且** 保持视频在列表中

### 需求：禁用下拉刷新
系统禁止在稍后再看页面使用下拉刷新功能，仅在页面加载时获取数据。

#### 场景：页面加载
- **当** 用户打开稍后再看页面
- **那么** 系统必须自动加载数据

#### 场景：尝试下拉刷新
- **当** 用户尝试下拉刷新
- **那么** 系统禁止触发刷新操作

### 需求：删除后自动刷新
系统必须在删除视频后自动刷新列表以显示最新状态。

#### 场景：删除成功后刷新
- **当** 删除视频操作成功
- **那么** 系统必须重新加载稍后再看列表
- **并且** 更新显示的内容

### 需求：优化列表布局
系统必须使用适合 iOS 设备的列表布局展示稍后再看列表。

#### 场景：iOS 设备显示
- **当** 在 iOS 设备上显示稍后再看列表
- **那么** 系统必须使用侧边栏样式的列表布局
- **并且** 保证视频信息完整清晰
- **并且** 支持流畅的滚动和交互
