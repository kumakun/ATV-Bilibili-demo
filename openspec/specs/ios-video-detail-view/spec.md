## 新增需求

### 需求：显示视频基本信息
系统必须在视频详情页展示视频的标题、封面图、UP主信息（头像、昵称）、发布时间和BVID。

#### 场景：成功加载视频信息
- **当** 用户进入视频详情页
- **那么** 系统显示视频标题、封面图、UP主头像和昵称、发布时间和BVID

#### 场景：加载失败显示错误
- **当** 视频信息加载失败
- **那么** 系统显示错误提示信息

### 需求：显示视频统计数据
系统必须展示视频的播放量、弹幕数、点赞数、投币数、收藏数和分享数。

#### 场景：展示统计数据
- **当** 视频信息加载完成
- **那么** 系统显示播放量、弹幕数、点赞数、投币数、收藏数和分享数

#### 场景：统计数据格式化
- **当** 统计数字大于1万
- **那么** 系统使用"万"为单位显示（如"12.5万"）

### 需求：显示视频简介
系统必须展示视频的完整简介文本，支持长文本展开和收起。

#### 场景：简介默认显示
- **当** 视频信息加载完成
- **那么** 系统显示视频简介的前3行内容

#### 场景：展开完整简介
- **当** 用户点击"展开"按钮
- **那么** 系统显示完整的视频简介内容

#### 场景：收起简介
- **当** 用户在展开状态点击"收起"按钮
- **那么** 系统收起简介，仅显示前3行

### 需求：加载状态指示
系统必须在加载视频信息时显示加载指示器。

#### 场景：显示加载状态
- **当** 系统正在加载视频信息
- **那么** 显示加载指示器

#### 场景：加载完成隐藏指示器
- **当** 视频信息加载完成
- **那么** 隐藏加载指示器并显示内容

### 需求：响应式布局
系统必须支持不同iOS设备尺寸的响应式布局。

#### 场景：iPhone竖屏显示
- **当** 用户在iPhone竖屏模式查看
- **那么** 系统使用单列布局展示所有信息

#### 场景：iPad显示
- **当** 用户在iPad查看
- **那么** 系统使用更宽松的布局，利用更大的屏幕空间
