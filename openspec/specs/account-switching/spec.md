# account-switching

## 目的

账号切换功能，在 iOS 端提供多账号管理界面，展示所有已登录账号，支持切换账号和添加新账号。

## 需求

### 需求：显示账号列表
系统必须显示所有已登录账号的列表，每个账号显示头像、用户名和当前激活状态。

#### 场景：加载账号列表
- **当** 用户打开账号切换页面
- **那么** 系统必须从账号管理器获取所有已登录账号
- **并且** 显示每个账号的头像和用户名

#### 场景：标记当前账号
- **当** 显示账号列表
- **那么** 系统必须标记当前激活的账号
- **并且** 使用视觉指示（如勾选图标或高亮）区分当前账号

### 需求：切换账号
系统必须支持用户切换到其他已登录的账号。

#### 场景：点击账号卡片
- **当** 用户点击某个非当前账号的卡片
- **那么** 系统必须将该账号设置为激活账号
- **并且** 关闭账号切换页面
- **并且** 重置应用主界面以反映新账号

#### 场景：点击当前账号
- **当** 用户点击当前激活的账号卡片
- **那么** 系统必须刷新当前账号的资料信息
- **并且** 关闭账号切换页面

### 需求：添加新账号
系统必须提供添加新账号的入口，允许用户登录更多账号。

#### 场景：显示添加账号按钮
- **当** 显示账号列表
- **那么** 系统必须在列表中显示"添加新账号"按钮或卡片

#### 场景：点击添加账号
- **当** 用户点击添加账号按钮
- **那么** 系统必须关闭账号切换页面
- **并且** 打开登录页面以添加新账号

### 需求：显示账号头像
系统必须为每个账号显示头像图片。

#### 场景：加载头像
- **当** 显示账号卡片
- **那么** 系统必须加载并显示该账号的头像
- **并且** 头像必须以圆形形式显示

#### 场景：头像加载失败
- **当** 账号头像加载失败
- **那么** 系统必须显示默认头像占位符

### 需求：显示账号信息
系统必须在账号卡片上显示用户名和其他关键信息。

#### 场景：显示用户名
- **当** 显示账号卡片
- **那么** 系统必须显示该账号的用户名

#### 场景：显示副标题
- **当** 显示账号卡片
- **那么** 系统可以显示账号的其他信息（如会员状态）

### 需求：响应账号更新
系统必须监听账号管理器的更新通知，并自动刷新账号列表。

#### 场景：账号信息变化
- **当** 账号信息发生变化（如新增、删除或更新账号）
- **那么** 系统必须自动重新加载账号列表
- **并且** 更新显示的内容

### 需求：关闭账号切换页面
系统必须提供关闭按钮，允许用户退出账号切换页面。

#### 场景：点击关闭按钮
- **当** 用户点击关闭按钮
- **那么** 系统必须关闭账号切换页面
- **并且** 返回到上一个页面

#### 场景：手势关闭
- **当** 用户使用下滑手势
- **那么** 系统必须支持手势关闭页面

### 需求：页面标题和说明
系统必须在账号切换页面顶部显示标题和简短说明。

#### 场景：显示标题
- **当** 打开账号切换页面
- **那么** 系统必须显示"账号管理"或"账号切换"标题

#### 场景：显示说明文本
- **当** 打开账号切换页面
- **那么** 系统必须显示说明文本（如"快速切换登录账号或添加新的账号"）

### 需求：优化 iOS 布局
系统必须使用适合 iOS 设备的布局展示账号列表。

#### 场景：iOS 设备显示
- **当** 在 iOS 设备上显示账号切换页面
- **那么** 系统必须使用适合移动设备的网格或列表布局
- **并且** 保证账号卡片清晰可点击
- **并且** 支持流畅的滚动
