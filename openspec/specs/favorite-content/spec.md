## 新增需求

### 需求：展示收藏夹中的视频列表
系统必须展示选定收藏夹中的所有视频，每个视频显示封面、标题、UP主信息和统计数据。

#### 场景：收藏夹有内容
- **当** 用户打开一个包含视频的收藏夹
- **那么** 系统必须以列表形式展示所有视频，显示封面图、标题、UP主名称和播放数据

#### 场景：收藏夹为空
- **当** 用户打开一个空收藏夹
- **那么** 系统必须显示空状态提示

### 需求：支持分页加载
系统必须支持收藏夹内容的分页加载，避免一次性加载过多数据。

#### 场景：滚动到底部自动加载
- **当** 用户滚动到列表底部且还有更多内容
- **那么** 系统必须自动加载下一页数据（每页 20 条）

#### 场景：没有更多内容
- **当** 用户滚动到底部且已加载全部内容
- **那么** 系统必须显示"已加载全部"提示

#### 场景：分页加载失败
- **当** 加载下一页数据失败
- **那么** 系统必须显示错误提示和重试选项，已加载的内容保持可见

### 需求：支持下拉刷新
系统必须支持下拉刷新收藏夹内容。

#### 场景：用户下拉刷新
- **当** 用户在收藏夹内容页面执行下拉刷新
- **那么** 系统必须清空当前列表并重新加载第一页数据

### 需求：点击视频跳转详情
系统必须支持用户点击视频卡片查看详情。

#### 场景：用户点击视频
- **当** 用户点击视频卡片
- **那么** 系统必须导航到该视频的详情页面（如详情页面尚未实现，则预留接口）

### 需求：处理 OGV 内容
系统必须正确处理收藏夹中的番剧、影视等 OGV（Occupationally Generated Video）内容。

#### 场景：收藏夹包含番剧内容
- **当** 收藏夹中包含番剧或影视内容
- **那么** 系统必须正确展示其封面、标题和集数信息

#### 场景：点击 OGV 内容
- **当** 用户点击番剧或影视内容
- **那么** 系统必须导航到对应的番剧/影视详情页

### 需求：显示加载状态
系统必须在加载和分页时提供清晰的状态反馈。

#### 场景：首次加载
- **当** 用户进入收藏夹内容页面
- **那么** 系统必须显示加载指示器直到首页数据加载完成

#### 场景：加载更多
- **当** 系统正在加载下一页数据
- **那么** 系统必须在列表底部显示加载指示器

### 需求：自适应布局
系统必须根据设备类型（iPhone/iPad）和屏幕方向提供自适应布局。

#### 场景：iPhone 竖屏
- **当** 在 iPhone 竖屏模式下查看
- **那么** 系统必须以单列布局展示视频列表

#### 场景：iPad 或 iPhone 横屏
- **当** 在 iPad 或 iPhone 横屏模式下查看
- **那么** 系统必须以多列网格布局展示视频列表
