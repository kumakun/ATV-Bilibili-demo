# 主导航规格说明

## 需求：主 Tab Bar 必须包含 3 个导航标签

应用的主导航栏必须包含且仅包含以下 3 个标签页，按照指定顺序排列：关注、收藏、我的。系统禁止在主导航中显示"推荐"、"热门"、"排行榜"标签。

#### 场景：应用启动时显示正确的 Tab 数量

- **当** 用户启动应用并成功登录
- **那么** 系统必须在主 Tab Bar 中显示恰好 3 个标签页

#### 场景：Tab 顺序正确

- **当** 用户查看主 Tab Bar
- **那么** 系统必须按以下顺序从左到右显示标签：
  1. 关注
  2. 收藏
  3. 我的

#### 场景：推荐热门排行榜标签不可见

- **当** 用户查看主 Tab Bar 中的所有标签
- **那么** 系统禁止显示"推荐"、"热门"、"排行榜"标签

## 需求：Tab 选中索引必须正确处理旧索引

当应用启动时，系统必须正确处理保存的 Tab 选中索引，即使该索引之前指向已移除的 Tab（推荐、热门、排行榜）。

#### 场景：启动时恢复有效的 Tab 选中状态

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值在有效范围内（0-2）
- **那么** 系统必须将 Tab Bar 切换到该索引对应的标签页

#### 场景：启动时处理旧的推荐热门排行榜 Tab 索引

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值为 0-2（旧的推荐、热门、排行榜索引）
- **那么** 系统必须默认显示"关注" Tab（新的索引 0）

#### 场景：启动时处理超出范围的索引

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值超出有效范围（>2）
- **那么** 系统必须默认显示"关注" Tab（索引 0）
