## 新增需求

### 需求：从关注页跳转
系统必须支持从关注页的视频卡片跳转到视频详情页。

#### 场景：点击视频卡片跳转
- **当** 用户在关注页点击任意视频卡片
- **那么** 系统导航到该视频的详情页

#### 场景：传递视频ID
- **当** 执行跳转操作
- **那么** 系统传递正确的视频aid和cid参数

### 需求：从收藏页跳转
系统必须支持从收藏页的视频列表跳转到视频详情页。

#### 场景：从收藏列表跳转
- **当** 用户在收藏页点击任意视频
- **那么** 系统导航到该视频的详情页

### 需求：从动态页跳转
系统必须支持从动态页的视频内容跳转到视频详情页。

#### 场景：点击动态中的视频
- **当** 用户在动态页点击视频类型的动态
- **那么** 系统导航到该视频的详情页

### 需求：导航返回
系统必须支持用户从视频详情页返回到上一页。

#### 场景：点击返回按钮
- **当** 用户点击导航栏的返回按钮
- **那么** 系统返回到上一级页面

#### 场景：保持上一页状态
- **当** 用户返回到上一页
- **那么** 系统保持上一页的滚动位置和内容状态

### 需求：深度链接支持
系统必须支持通过URL Scheme打开视频详情页。

#### 场景：处理视频链接
- **当** 用户通过外部链接打开视频
- **那么** 系统解析链接并直接跳转到对应的视频详情页

#### 场景：无效链接处理
- **当** 接收到无效的视频链接
- **那么** 系统显示错误提示

### 需求：页面转场动画
系统必须使用标准的iOS转场动画进行页面切换。

#### 场景：导航转场
- **当** 进入或退出视频详情页
- **那么** 系统使用平滑的push/pop转场动画

### 需求：进入视频详情页时必须隐藏主 Tab Bar
系统必须在用户进入视频详情页时隐藏主 Tab Bar，并在用户离开视频详情页返回上一级后恢复显示主 Tab Bar。

#### 场景：从关注页进入视频详情页
- **当** 用户在关注页点击视频卡片并进入视频详情页
- **那么** 系统导航到对应的视频详情页
- **那么** 系统隐藏主 Tab Bar

#### 场景：从收藏页进入视频详情页
- **当** 用户从收藏夹列表进入任意视频详情页
- **那么** 系统导航到对应的视频详情页
- **那么** 系统隐藏主 Tab Bar

#### 场景：从视频详情页返回上一级
- **当** 用户从视频详情页返回原列表页
- **那么** 系统恢复显示主 Tab Bar
- **那么** 系统保持原列表页的内容状态和滚动位置
