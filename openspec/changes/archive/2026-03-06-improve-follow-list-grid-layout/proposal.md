## 为什么

当前关注列表页在 iPad 上沿用偏窄的单一移动端布局，横向空间利用率不足，导致同屏可见内容少、浏览效率低。随着 iPad 端体验持续补齐，需要为不同尺寸设备定义更清晰的网格展示要求，让关注列表在 iPhone 与 iPad 上都具备稳定且高效的浏览体验。

## 变更内容

- 调整关注 UP 主列表的布局需求，明确在 iPhone 上使用 2 列展示。
- 调整关注 UP 主列表的布局需求，明确在 iPad 上根据可用宽度使用多列网格展示。
- 保持现有分页加载、错误处理、空状态和点击进入 UP 主空间等行为不变，仅增强不同尺寸设备下的列表展示规则。

## 功能 (Capabilities)

### 新增功能

无

### 修改功能

- `follow-ups-list`: 更新关注列表在 iPhone 与 iPad 上的网格布局要求，明确不同设备尺寸下的列数策略与卡片展示约束。

## 影响

- iOS 端关注列表相关 SwiftUI 页面与网格布局实现
- 关注列表卡片尺寸、间距和自适应列数策略
- 关注列表现有 UI 验收标准与对应规范文档
