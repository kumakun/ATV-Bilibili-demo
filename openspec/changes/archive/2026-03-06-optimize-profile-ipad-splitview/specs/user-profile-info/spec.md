# user-profile-info

## 移除需求

### 需求：显示统计数据

**Reason**: 统计数据（关注数、粉丝数、获赞数）在个人中心页面的使用价值较低，占用了宝贵的垂直空间。移除这些数据可以简化页面结构，让用户更关注核心功能。在 iPad 上使用 SplitView 布局后，空间利用需要更加高效。

**Migration**: 
- 删除 `UserStatsView` 和 `StatItem` 组件
- 从 `ProfileViewModel` 中移除 `followingCount`、`followerCount`、`likesCount` 属性
- 从数据模型中移除对应的统计数据字段（`UserProfile.following`、`UserProfile.follower`、`UserProfile.likes`）
- 如果后续需要查看统计数据，可以考虑在用户个人主页或账号管理页面中提供入口

#### 场景：显示统计数值

#### 场景：未登录时的统计

### 需求：响应账号更新

**Reason**: 修改需求以反映移除统计数据后的行为变化。

**Migration**: 更新场景描述，不再提及统计数据的更新。

## 修改需求

### 需求：响应账号更新
系统必须监听账号管理器的更新通知，并在账号信息变化时自动刷新显示的用户信息。

#### 场景：账号切换时更新
- **当** 用户切换到另一个账号
- **那么** 系统必须自动更新显示的头像、用户名和签名

#### 场景：账号登出时更新
- **当** 用户登出账号
- **那么** 系统必须清除显示的用户信息并显示未登录状态
