## 新增需求

### 需求：展示二维码登录入口

iOS 应用必须在未登录状态下展示二维码登录的入口按钮。

#### 场景：未登录用户看到登录入口
- **当** 用户打开应用且未登录
- **那么** 系统必须显示"扫码登录"按钮

#### 场景：点击登录按钮打开二维码界面
- **当** 用户点击"扫码登录"按钮
- **那么** 系统必须以 Sheet 弹窗形式展示二维码登录界面

### 需求：生成登录二维码

系统必须向哔哩哔哩登录服务器请求二维码，并在界面上展示。

#### 场景：成功获取二维码
- **当** 二维码登录界面打开
- **那么** 系统必须调用 `requestLoginQR` API 获取 authCode 和 URL
- **并且** 系统必须将 URL 转换为二维码图像
- **并且** 系统必须在界面上显示生成的二维码

#### 场景：二维码生成失败
- **当** API 请求失败
- **那么** 系统必须显示错误提示
- **并且** 系统必须提供"重新生成二维码"按钮

### 需求：轮询验证登录状态

系统必须定期检查二维码的扫码和验证状态。

#### 场景：启动轮询验证
- **当** 二维码生成成功
- **那么** 系统必须每 4 秒调用一次 `verifyLoginQR` API
- **并且** 系统必须传递 authCode 作为参数

#### 场景：扫码等待中
- **当** API 返回状态码 86039（等待扫码）
- **那么** 系统必须保持二维码显示
- **并且** 系统必须继续轮询

#### 场景：二维码过期
- **当** API 返回状态码 86038（二维码过期）
- **那么** 系统必须停止轮询
- **并且** 系统必须自动重新生成新的二维码

#### 场景：登录成功
- **当** API 返回成功状态并包含 token 和 cookies
- **那么** 系统必须停止轮询
- **并且** 系统必须将 token 和 cookies 传递给账号管理模块进行注册
- **并且** 系统必须关闭二维码登录界面
- **并且** 系统必须更新应用的登录状态为已登录

#### 场景：超过最大轮询次数
- **当** 轮询次数超过 200 次（约 800 秒）
- **那么** 系统必须停止轮询
- **并且** 系统必须显示超时提示

### 需求：手动刷新二维码

用户必须能够手动触发二维码刷新。

#### 场景：用户点击刷新按钮
- **当** 用户点击"重新生成二维码"按钮
- **那么** 系统必须停止当前的轮询
- **并且** 系统必须重新调用 `requestLoginQR` API
- **并且** 系统必须生成并显示新的二维码
- **并且** 系统必须重新启动轮询验证

### 需求：取消登录操作

用户必须能够取消登录流程并返回。

#### 场景：用户取消登录
- **当** 用户在二维码登录界面点击"取消"按钮
- **那么** 系统必须停止轮询
- **并且** 系统必须关闭二维码登录界面
- **并且** 系统必须返回到未登录的主界面

### 需求：显示登录说明

界面必须向用户展示清晰的登录指引。

#### 场景：显示扫码说明
- **当** 二维码登录界面显示
- **那么** 系统必须显示文本："请使用哔哩哔哩客户端扫描二维码登录"
- **并且** 说明文本必须清晰可读

### 需求：视图生命周期管理

系统必须正确管理二维码登录界面的生命周期。

#### 场景：界面消失时清理资源
- **当** 二维码登录界面被关闭或消失
- **那么** 系统必须停止所有正在进行的轮询计时器
- **并且** 系统必须清理二维码图像资源
- **并且** 系统必须取消未完成的网络请求
