## 新增需求

### 需求：定义 FavListData 模型
系统必须定义 FavListData 数据模型用于表示收藏夹信息。

#### 场景：模型包含基本字段
- **当** 系统解析收藏夹列表 API 响应
- **那么** FavListData 模型必须包含以下字段：id（收藏夹ID）、title（收藏夹名称）、mediaCount（内容数量，可选）

#### 场景：模型支持区分自建和订阅
- **当** 系统需要区分收藏夹类型
- **那么** FavListData 模型必须包含 isCreatedBySelf（布尔值）字段标识是否为用户自建

#### 场景：模型支持订阅收藏夹的UP主信息
- **当** 收藏夹为订阅的他人收藏夹
- **那么** FavListData 模型必须包含 mid（UP主ID，可选）字段

### 需求：定义 FavData 模型
系统必须定义 FavData 数据模型用于表示收藏的视频信息。

#### 场景：模型包含视频基本信息
- **当** 系统解析收藏视频列表 API 响应
- **那么** FavData 模型必须包含以下字段：id（视频aid）、title（标题）、cover（封面URL）、intro（简介，可选）、duration（时长）

#### 场景：模型包含UP主信息
- **当** 系统需要展示视频UP主
- **那么** FavData 模型必须包含嵌套的 upper 对象，包含 mid、name 和 face 字段

#### 场景：模型包含统计信息
- **当** 系统需要展示视频数据
- **那么** FavData 模型必须包含 cnt_info 对象，包含 play（播放数）和 collect（收藏数）字段

#### 场景：模型支持 OGV 内容
- **当** 收藏内容为番剧或影视
- **那么** FavData 模型必须包含 ogv 对象（可选），包含 season_id 字段用于标识番剧

### 需求：实现 Codable 协议
系统必须让所有数据模型实现 Codable 协议以支持 JSON 解析。

#### 场景：解析 API JSON 响应
- **当** 系统接收到 B站 API 的 JSON 响应
- **那么** 数据模型必须能够自动解码为对应的 Swift 对象

#### 场景：字段名映射
- **当** API 字段名与 Swift 命名规范不一致（如使用下划线）
- **那么** 系统必须使用 CodingKeys 枚举进行正确的字段映射

### 需求：处理可选字段
系统必须正确处理 API 返回的可选字段，避免解析失败。

#### 场景：缺失可选字段
- **当** API 响应中缺少某些可选字段（如 intro、mediaCount）
- **那么** 系统必须将其解析为 nil，而不是抛出解析错误

#### 场景：空值字段
- **当** API 返回 null 值
- **那么** 系统必须正确处理为 Optional.none

### 需求：实现模型扩展属性
系统可以为模型添加计算属性以提供便利方法。

#### 场景：格式化时长
- **当** UI 需要展示视频时长
- **那么** FavData 模型应提供 durationText 计算属性，将秒数转换为"mm:ss"或"hh:mm:ss"格式

#### 场景：格式化播放数
- **当** UI 需要展示播放数
- **那么** 模型应提供格式化方法，将大数字转换为易读格式（如"1.2万"）
