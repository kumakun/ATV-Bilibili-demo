## 修改需求

### 需求:视频播放 seek 行为
系统必须在用户拖动进度条后从目标时间点继续播放视频，禁止从视频起点重新开始播放。无论 SIDX segment 索引是否成功加载，seek 行为都必须正确工作。

#### 场景:SIDX 可用时 seek
- **当** SIDX box 成功下载并解析，用户拖动进度条到目标时间
- **那么** 视频从目标时间点开始播放，误差不超过最近关键帧距离

#### 场景:SIDX 不可用时 seek（回退路径）
- **当** SIDX box 下载失败进入回退路径，用户拖动进度条到目标时间
- **那么** 视频从目标时间点附近开始播放，禁止从视频起点重新开始

#### 场景:回退播放列表使用正确时长
- **当** 系统生成回退 HLS 播放列表（无 SIDX）
- **那么** 播放列表中 EXTINF 和 TARGETDURATION 的值必须等于视频实际时长（秒），禁止使用硬编码的 1 秒值

## 新增需求

### 需求:SIDX 下载重试
系统必须在 SIDX 下载失败时进行重试，最多尝试 3 次（含首次），每次请求超时不超过 10 秒。仅当所有重试均失败后才进入回退路径。

#### 场景:首次下载失败后重试成功
- **当** SIDX 首次下载因网络错误失败，第二次重试成功
- **那么** 系统使用成功获取的 SIDX 数据生成详细播放列表，不进入回退路径

#### 场景:所有重试均失败
- **当** SIDX 三次下载尝试均失败
- **那么** 系统进入回退路径生成简单播放列表，使用视频实际时长作为 segment 时长

### 需求:精确时长 asset 配置
系统必须在创建 AVURLAsset 时将 `AVURLAssetPreferPreciseDurationAndTimingKey` 设为 `true`，以确保 seek 定位精确。

#### 场景:asset 创建配置
- **当** 系统为 DASH 视频流创建 AVURLAsset
- **那么** asset 的 options 中 `AVURLAssetPreferPreciseDurationAndTimingKey` 必须为 `true`
