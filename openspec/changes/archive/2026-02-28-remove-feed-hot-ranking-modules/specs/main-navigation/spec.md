# 主导航规格说明 - 增量更新

## 修改需求

### 需求：主 Tab Bar 必须包含 3 个导航标签

应用的主导航栏必须包含且仅包含以下 3 个标签页，按照指定顺序排列：关注、收藏、我的。系统禁止在主导航中显示"直播"、"推荐"、"热门"或"排行榜"标签。

#### 场景：应用启动时显示正确的 Tab 数量

- **当** 用户启动应用并成功登录
- **那么** 系统必须在主 Tab Bar 中显示恰好 3 个标签页

#### 场景：Tab 顺序正确

- **当** 用户查看主 Tab Bar
- **那么** 系统必须按以下顺序从左到右显示标签：
  1. 关注
  2. 收藏
  3. 我的

#### 场景：已移除标签不可见

- **当** 用户查看主 Tab Bar 中的所有标签
- **那么** 系统禁止显示"直播"、"推荐"、"热门"或"排行榜"标签

### 需求：Tab 选中索引必须正确处理

当应用启动时，系统必须正确处理保存的 Tab 选中索引，包括处理之前指向已移除标签页的旧索引值（推荐/热门/排行榜/直播）。

#### 场景：启动时恢复有效的 Tab 选中状态

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值在有效范围内（0-2）
- **那么** 系统必须将 Tab Bar 切换到该索引对应的标签页

#### 场景：启动时处理旧的推荐/热门/排行榜 Tab 索引

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值为 0、1 或 2（旧的"推荐"、"热门"或"排行榜" Tab 索引）
- **那么** 系统必须默认显示"关注" Tab（新的索引 0）

#### 场景：启动时处理旧的关注/收藏/我的 Tab 索引

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值为 3、4 或 5（旧的"关注"、"收藏"或"我的" Tab 索引）
- **那么** 系统必须将索引映射到新的位置：3→0（关注）、4→1（收藏）、5→2（我的）

#### 场景：启动时处理超出范围的索引

- **当** 应用启动且 UserDefaults 中保存的 selectedIndex 值超出有效范围（>5）
- **那么** 系统必须默认显示"关注" Tab（索引 0）

## 移除需求

### 需求：推荐、热门、排行榜标签页功能

**Reason**: 简化应用导航结构，聚焦于关注、收藏和个人中心三个核心功能。推荐、热门、排行榜功能及其相关代码已从应用中完全移除。

**Migration**: 
- 用户应使用"关注"标签页查看已关注 UP 主的内容
- 用户可使用"收藏"标签页访问已保存的内容
- 无直接替代推荐、热门、排行榜功能的方案
