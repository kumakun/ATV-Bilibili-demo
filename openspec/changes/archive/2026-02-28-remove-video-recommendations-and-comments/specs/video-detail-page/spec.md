# 视频详情页规范

## 移除需求

### 需求：显示推荐视频列表
**Reason**: 简化界面，减少用户注意力分散，提升对当前视频内容的专注度
**Migration**: 用户需要返回首页或使用搜索功能来发现其他视频内容

### 需求：显示热门评论
**Reason**: 简化界面，减少用户注意力分散，提升对当前视频内容的专注度
**Migration**: 用户若需要查看评论，可通过其他专门的评论入口或功能访问

## 修改需求

### 需求：视频详情页必须展示核心视频信息
视频详情页必须专注展示当前视频的核心信息，包括但不限于标题、UP主信息、播放数据、视频描述、分P列表和合集信息。视频详情页禁止展示推荐视频和评论内容。

#### 场景：用户打开视频详情页
- **当** 用户打开一个视频的详情页
- **那么** 系统必须显示视频标题、封面、UP主信息、播放按钮
- **那么** 系统必须显示播放量、弹幕数、上传时间等统计信息
- **那么** 系统必须显示视频描述内容
- **那么** 系统禁止显示推荐视频列表
- **那么** 系统禁止显示评论内容

#### 场景：多P视频展示
- **当** 用户打开一个包含多个分P的视频详情页
- **那么** 系统必须显示所有分P的列表
- **那么** 系统必须允许用户选择任意分P进行播放

#### 场景：合集视频展示
- **当** 用户打开一个属于合集的视频详情页
- **那么** 系统必须显示合集信息和合集内的其他视频
- **那么** 系统必须允许用户导航到合集内的其他视频

#### 场景：用户交互功能保持可用
- **当** 用户在视频详情页进行操作
- **那么** 系统必须允许用户点击播放按钮开始播放
- **那么** 系统必须允许用户进行点赞、投币、收藏等互动操作
- **那么** 系统必须允许用户关注UP主
- **那么** 系统必须允许用户查看UP主空间
