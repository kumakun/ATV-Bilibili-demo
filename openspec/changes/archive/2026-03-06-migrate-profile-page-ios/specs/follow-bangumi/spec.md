## 新增需求

### 需求：分类展示追番追剧
系统必须将追番追剧分为"番剧"和"影视"两个分类标签，用户可以切换查看不同类型的内容。

#### 场景：显示分类标签
- **当** 用户打开追番追剧页面
- **那么** 系统必须显示"番剧"和"影视"两个分类标签

#### 场景：切换分类
- **当** 用户点击某个分类标签
- **那么** 系统必须切换到对应的内容列表
- **并且** 高亮显示当前选中的标签

### 需求：显示番剧列表
系统必须显示用户关注的番剧列表，包括封面图、标题、观看进度和最新集信息。

#### 场景：加载番剧列表
- **当** 用户选择"番剧"分类
- **那么** 系统必须从服务器获取番剧列表（type=1）
- **并且** 显示每个番剧的封面、标题和进度信息

#### 场景：显示最新集信息
- **当** 番剧有最新集更新
- **那么** 系统必须显示最新集的序号（index_show）
- **并且** 使用最新集封面作为显示封面

#### 场景：显示更新标记
- **当** 番剧在今天有新集更新
- **那么** 系统必须显示"更新了!"标记
- **并且** 突出显示该番剧

### 需求：显示影视列表
系统必须显示用户关注的影视列表，包括封面图、标题、观看进度和最新集信息。

#### 场景：加载影视列表
- **当** 用户选择"影视"分类
- **那么** 系统必须从服务器获取影视列表（type=2）
- **并且** 显示每个影视作品的封面、标题和进度信息

### 需求：分页加载
系统必须支持分页加载追番追剧列表，每页加载 24 条数据。

#### 场景：首次加载
- **当** 用户打开某个分类
- **那么** 系统必须加载第一页数据（24 条）

#### 场景：加载更多
- **当** 用户滚动到列表底部
- **并且** 还有更多数据
- **那么** 系统必须自动加载下一页数据

#### 场景：加载失败
- **当** 数据加载失败
- **那么** 系统必须显示错误信息
- **并且** 允许用户重试

### 需求：显示观看进度
系统必须在每个番剧/影视卡片上显示用户的观看进度信息。

#### 场景：显示进度文本
- **当** 番剧有观看进度信息
- **那么** 系统必须在封面下方显示进度文本（如"已看到第 5 集"）

#### 场景：无进度信息
- **当** 番剧没有观看进度
- **那么** 系统必须显示为空或默认文本

### 需求：导航到详情页
系统必须支持用户点击番剧/影视卡片后导航到详情页面。

#### 场景：点击番剧卡片
- **当** 用户点击某个番剧或影视卡片
- **那么** 系统必须打开该作品的详情页面
- **并且** 传递 season_id 参数

### 需求：优化网格布局
系统必须使用网格布局展示番剧和影视列表，适配 iOS 移动设备屏幕。

#### 场景：iOS 设备显示
- **当** 在 iOS 设备上显示列表
- **那么** 系统必须使用适合移动设备的列数
- **并且** 每个卡片必须清晰显示封面和文本信息
