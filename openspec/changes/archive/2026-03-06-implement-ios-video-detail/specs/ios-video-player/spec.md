## 新增需求

### 需求：视频播放功能
系统必须支持在线视频播放，使用AVPlayer作为播放引擎。

#### 场景：开始播放视频
- **当** 用户点击播放按钮
- **那么** 系统加载视频URL并开始播放

#### 场景：播放失败处理
- **当** 视频URL加载失败或播放出错
- **那么** 系统显示错误提示并允许用户重试

### 需求：播放控制
系统必须提供标准的视频播放控制功能，包括播放、暂停、进度拖动和全屏。

#### 场景：播放和暂停
- **当** 用户点击播放/暂停按钮
- **那么** 系统切换视频播放状态

#### 场景：进度拖动
- **当** 用户拖动进度条
- **那么** 系统跳转到对应的播放位置

#### 场景：全屏播放
- **当** 用户点击全屏按钮
- **那么** 系统进入全屏播放模式

### 需求：画质选择
系统必须支持多个清晰度选项，用户可以手动切换画质。

#### 场景：显示可用画质
- **当** 用户点击画质选择按钮
- **那么** 系统显示所有可用的画质选项（如流畅、高清、超清等）

#### 场景：切换画质
- **当** 用户选择不同的画质
- **那么** 系统切换到新画质并继续当前播放位置

### 需求：播放进度记忆
系统必须记录用户的播放进度，下次进入时继续播放。

#### 场景：保存播放进度
- **当** 用户退出视频播放
- **那么** 系统保存当前播放进度

#### 场景：恢复播放进度
- **当** 用户再次进入同一视频且播放进度大于5秒
- **那么** 系统询问是否从上次位置继续播放

### 需求：播放器UI集成
系统必须使用SwiftUI的VideoPlayer组件展示播放器界面。

#### 场景：显示播放器
- **当** 视频详情页加载完成
- **那么** 系统在页面顶部展示视频播放器

#### 场景：播放器尺寸适配
- **当** 在不同设备上显示
- **那么** 播放器保持16:9的宽高比
