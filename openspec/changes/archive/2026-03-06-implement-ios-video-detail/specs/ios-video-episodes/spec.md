## 新增需求

### 需求：显示视频分P列表
系统必须展示多P视频的所有分集信息。

#### 场景：显示分P列表
- **当** 视频包含多个分P
- **那么** 系统显示分P选集列表，包含每一P的标题和时长

#### 场景：单P视频不显示
- **当** 视频只有1个分P
- **那么** 系统不显示分P选集区域

### 需求：分P切换
系统必须支持用户切换到不同的分P观看。

#### 场景：选择其他分P
- **当** 用户点击分P列表中的某一集
- **那么** 系统加载该分P的视频并更新页面信息

#### 场景：高亮当前分P
- **当** 显示分P列表
- **那么** 系统高亮显示当前正在播放的分P

### 需求：显示合集信息
系统必须支持展示视频所属的合集。

#### 场景：显示合集列表
- **当** 视频属于某个合集
- **那么** 系统显示合集名称和合集中的其他视频

#### 场景：合集视频跳转
- **当** 用户点击合集中的其他视频
- **那么** 系统跳转到对应视频的详情页

### 需求：连续播放
系统必须支持多P视频的自动连播功能。

#### 场景：自动播放下一P
- **当** 当前分P播放结束且存在下一P
- **那么** 系统自动加载并播放下一P

#### 场景：最后一P播放完成
- **当** 最后一个分P播放结束
- **那么** 系统停止播放并显示播放完成状态

### 需求：分P信息展示
系统必须显示每个分P的基本信息。

#### 场景：显示分P标题
- **当** 展示分P列表
- **那么** 每个分P显示其标题

#### 场景：显示分P时长
- **当** 展示分P列表
- **那么** 每个分P显示其视频时长
