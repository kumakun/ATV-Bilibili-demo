## 新增需求

### 需求：显示观看历史列表
系统必须显示用户的视频观看历史记录，每条记录包括视频封面、标题、UP 主名称和观看时间等信息。

#### 场景：加载历史记录
- **当** 用户打开历史记录页面
- **那么** 系统必须从服务器获取用户的观看历史
- **并且** 按时间倒序显示历史记录

#### 场景：加载失败
- **当** 获取历史记录失败
- **那么** 系统必须显示错误提示
- **并且** 允许用户重试

#### 场景：空历史记录
- **当** 用户没有任何观看历史
- **那么** 系统必须显示空状态提示

### 需求：显示视频信息
系统必须在每条历史记录中显示视频的基本信息，包括封面图、标题和 UP 主信息。

#### 场景：显示完整信息
- **当** 显示历史记录项
- **那么** 系统必须显示视频封面图
- **并且** 显示视频标题
- **并且** 显示 UP 主名称

#### 场景：封面加载失败
- **当** 视频封面加载失败
- **那么** 系统必须显示默认占位图

### 需求：显示观看进度
系统必须在历史记录中显示用户的观看进度信息。

#### 场景：显示观看位置
- **当** 历史记录包含观看进度
- **那么** 系统必须显示用户观看到的时间点或进度百分比
- **并且** 在封面上叠加进度条指示

#### 场景：无进度信息
- **当** 历史记录没有进度信息
- **那么** 系统可以不显示进度指示

### 需求：继续播放
系统必须支持用户点击历史记录后从上次观看位置继续播放视频。

#### 场景：点击历史记录
- **当** 用户点击某条历史记录
- **那么** 系统必须打开视频详情页面
- **并且** 传递视频的 aid 和 cid 参数
- **并且** 自动从上次观看位置开始播放

### 需求：自动刷新
系统必须在页面显示时自动刷新历史记录，确保数据是最新的。

#### 场景：页面显示时刷新
- **当** 用户进入历史记录页面
- **那么** 系统必须重新加载最新的历史记录

#### 场景：从其他页面返回
- **当** 用户从视频播放页面返回历史记录
- **那么** 系统必须刷新列表以反映新的观看记录

### 需求：列表布局优化
系统必须使用适合 iOS 设备的列表布局展示历史记录。

#### 场景：iOS 设备显示
- **当** 在 iOS 设备上显示历史记录
- **那么** 系统必须使用适合移动设备的列表样式
- **并且** 保证内容清晰可读
- **并且** 支持流畅的滚动体验

### 需求：显示观看时间
系统必须显示每条历史记录的观看时间信息，如"今天"、"昨天"、"3 天前"等。

#### 场景：显示相对时间
- **当** 显示历史记录
- **那么** 系统必须显示相对时间描述（如"今天"、"昨天"）

#### 场景：显示具体时间
- **当** 历史记录较久远
- **那么** 系统必须显示具体日期（如"2026-03-01"）
